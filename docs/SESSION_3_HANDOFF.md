## Cached JSON location (single source of truth caveat)

Next.js / Turbopack refuses to resolve static imports from outside the project root (`frontend/`). The three analyses live canonically at `scripts/cached/` but the webpack resolver would not pick them up via `../../../scripts/cached/*.json`. The build failed with `Module not found: Can't resolve '../../../scripts/cached/high_donotfind.json'`.

Fix: copied the three JSONs into `frontend/components/live-demo/cached/`. The component imports from the co-located copy.

Tradeoff: if `scripts/cached/` is regenerated by the labeling pipeline, the frontend copy will drift. Recommend adding a one-liner to `cache_gallery.py` (or a prebuild npm script) to sync both, or pinning these three as demo constants. Did not add that sync since it would touch pipeline code out of session scope.

## Embed loading strategy

- Each `<TikTokEmbed>` mounts an `IntersectionObserver` on its container with a `250px` root margin so the embed boot starts a little before the section is visible.
- The first card to cross the threshold calls `loadEmbedScript()`, which injects `https://www.tiktok.com/embed.js` once via a module-scoped `Promise` singleton. Cards that observe later await the same promise and never re-inject.
- TikTok's `embed.js` scans the DOM for `blockquote.tiktok-embed` nodes on load and replaces each with the real iframe. The three blockquotes are rendered at SSR so they are in the DOM before the script initializes.
- If the script fails to load (CSP, network block, adblock), the card swaps in `EmbedFallback`, which shows a 9:16 placeholder with an explicit "Watch on TikTok" button. The page does not break.
- The iframe that TikTok injects has its own width and min-width styles, so the container uses Tailwind v4 arbitrary variants `[&_iframe]:!max-w-full [&_iframe]:!min-w-0` to prevent iPhone-SE-width overflow.
- No autoplay. Audio is muted until the user taps the embed.

If that feels too long on a 27-inch monitor, the alternative would be a 2x2-minus-1 grid, but that risks breaking the narrative flow. I would not change this without a design review.

## Things worth knowing

- The cached JSON duplication (`scripts/cached/` vs `frontend/components/live-demo/cached/`) is real technical debt. If the labeling pipeline rewrites any of those three analyses, the frontend copy silently goes stale. Cheapest long-term fix: a prebuild npm script that copies the three canonical files into the frontend tree.
- `tiktok-embed.tsx` guards against the iframe overflow that TikTok's default styling tries to force (`min-width: 325px`). If a future TikTok update changes the iframe's element class, the `[&_iframe]` overrides may need to be revisited.
- The "Watch on TikTok" link is both a primary affordance (shown under every embed) and the fallback CTA (shown inside `EmbedFallback`). Intentional redundancy, accessibility-first.
- The card eyebrow is the uploader handle (`@donotfindthisaccountever`, etc). Not part of the locked copy; added as factual orienting context.
